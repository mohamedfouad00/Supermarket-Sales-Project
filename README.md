This project transforms transactional supermarket data into actionable business intelligence that drives revenue growth and operational excellence.

**Key Metrics:**
- **$322,966.75** Total Revenue (Jan–Mar 2019)

- **4.76%** Net Profit Margin
- **5,510** Transactions Analyzed
- **6.97 / 10** Average Customer Rating
- **62%** of Profit Generated by Large Orders

## SQL Queries

### 1. Branch Profitability Analysis
```sql
SELECT
    db.Branch,
    db.City,
    COUNT(fs.[Invoice ID])          AS Orders,
    SUM(fs.Total)                   AS Revenue,
    SUM(fs.[gross income])          AS Profit,
    ROUND(SUM(fs.[gross income]) 
          / SUM(fs.Total) * 100, 2) AS Profit_Margin_Pct,
    ROUND(AVG(fs.Total), 2)         AS Avg_Order_Value
FROM Fact_Sales fs
INNER JOIN Dim_Branches db
    ON fs.BranchKey = db.BranchKey
GROUP BY db.Branch, db.City
ORDER BY Profit DESC;
```

**Output:** Naypyitaw (5,265) leads. Yangon highest order count (1,859) but Naypyitaw achieves more profit on fewer orders.

---

### 2. Customer Segmentation Analysis
```sql
SELECT
    dc.CustomerType,
    dc.Gender,
    COUNT(fs.[Invoice ID])          AS Orders,
    SUM(fs.Total)                   AS Revenue,
    SUM(fs.[gross income])          AS Profit,
    ROUND(AVG(fs.[gross income]), 2) AS Avg_Profit_Per_Order
FROM Fact_Sales fs
INNER JOIN Dim_Customers dc
    ON fs.CustomerKey = dc.CustomerKey
GROUP BY dc.CustomerType, dc.Gender
ORDER BY Profit DESC;
```

**Output:** Female Members: 7,994 profit (highest). Members outperform Normal by 3.5%. Females outperform Males by 8.2%.

---

### 3. Product Line Performance
```sql
SELECT
    dp.ProductLine,
    COUNT(fs.[Invoice ID])          AS Orders,
    SUM(fs.Total)                   AS Revenue,
    SUM(fs.[gross income])          AS Profit,
    ROUND(SUM(fs.[gross income]) 
          / SUM(fs.Total) * 100, 2) AS Profit_Margin_Pct,
    ROW_NUMBER() OVER (ORDER BY SUM(fs.[gross income]) DESC)
                                    AS Rank_By_Profit
FROM Fact_Sales fs
INNER JOIN Dim_Products dp
    ON fs.ProductKey = dp.ProductKey
GROUP BY dp.ProductLine
ORDER BY Profit DESC;
```

**Output:** Food & Beverages leads (2,674). All 6 lines within 14% profit range — low category concentration risk.

---

### 4. Order Size Profitability
```sql
SELECT
    fs.order_status,
    COUNT(fs.[Invoice ID])          AS Orders,
    SUM(fs.Total)                   AS Revenue,
    SUM(fs.[gross income])          AS Profit,
    ROUND(SUM(fs.[gross income]) 
          / SUM(fs.Total) * 100, 2) AS Profit_Margin_Pct,
    ROUND(SUM(fs.[gross income]) 
          * 100.0 / SUM(SUM(fs.[gross income]))
          OVER (), 2)               AS Profit_Share_Pct
FROM Fact_Sales fs
GROUP BY fs.order_status
ORDER BY Profit DESC;
```

**Output:** Large Orders: 62% of profit. Small Orders: 11%. Conversion from Medium to Large is key lever.

---

### 5. Female Member Segmentation (High-Value Target)
```sql
SELECT
    dc.CustomerType,
    dc.Gender,
    COUNT(fs.[Invoice ID])          AS Orders,
    SUM(fs.Total)                   AS Revenue,
    SUM(fs.[gross income])          AS Profit,
    ROUND(AVG(fs.Rating), 2)        AS Avg_Rating
FROM Fact_Sales fs
INNER JOIN Dim_Customers dc
    ON fs.CustomerKey = dc.CustomerKey
WHERE dc.CustomerType = 'Member'
  AND dc.Gender IN ('Female', 'Male')
GROUP BY dc.CustomerType, dc.Gender
ORDER BY Profit DESC;
```

**Output:** Female Members lead all segments. Priority target for loyalty program expansion and retention initiatives.

## Business Recommendations

### Immediate Actions (High Priority)

1. **Anchor strategy in large orders**  
   Large orders drive 62% of profit. Incentivize bulk purchases with bundle pricing and spend-threshold discounts.

2. **Leverage loyalty programs**  
   Members outperform Normal customers (7,820 vs 7,559 profit). Expand member benefits to convert more customers.

3. **Target female customers**  
   Female segment contributes 8.2% more profit. Tailor promotions, product placement, and marketing accordingly.

4. **Balance product line focus**  
   Food & Beverages leads, but maintain diversification across all 6 lines to reduce category-level risk.

5. **Improve rating experience**  
   Enhance service quality, particularly at high-spend touchpoints. Target: move more transactions from low/medium to high ratings.

6. **Replicate Naypyitaw's model**  
   Naypyitaw achieves highest profit despite mid-range order count. Study and transfer its operational practices to Yangon and Mandalay.

## Getting Started

### Prerequisites
- SQL Server 2019 or higher
- Power BI Desktop (free or paid)
- Excel 2016 or higher
- Git (optional)

### Installation

1. **Clone the repository**
```bash
git clone https://github.com/mohamedfouad00/Supermarket-Sales-Analytics.git
cd Supermarket-Sales-Analytics
```

2. **Set up SQL Server**
   - Create the star schema tables using scripts in `SQL/Schema_Creation.sql`
   - Load the transactional data into `Fact_Sales`
   - Load dimension tables: `Dim_Branches`, `Dim_Customers`, `Dim_Products`, `Dim_Date`

3. **Open Power BI Dashboards**
   - Connect to your SQL Server instance
   - Open `.pbix` files in Power BI Desktop
   - Refresh data connections

4. **Open Excel Dashboard**
   - Open the Excel workbook
   - Enable Power Query connections
   - Refresh pivot tables and slicers

### Running Queries

```sql
-- Connect to your database
USE [SupermarketSalesDB];

-- Execute analytical queries
-- Branch Performance
EXEC sp_branch_profitability;

-- Customer Segmentation
EXEC sp_customer_segmentation;

-- Product Performance
EXEC sp_product_analysis;
```
